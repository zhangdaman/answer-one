# 首答 AnswerOne

面向 **GEO（生成式引擎优化 / Generative Engine Optimization）** 的内容工作台：帮品牌诊断自己在各大 AI 平台里的「可被搜到 / 被提及」程度，并据此生成与分发内容、持续复诊。

> 当前为**原型 + 设计阶段**：高保真可交互原型与产品/接口设计已完成，后端目前为骨架（`pycore`），尚未实现业务逻辑。

---

## 它解决什么

用户在豆包、千问、DeepSeek、文心、Kimi、腾讯元宝等 AI 助手里提本地化问题时（如「杭州有什么好喝的奶茶？」），品牌往往根本不被提及。AnswerOne 把这件事变成一条可量化、可执行的闭环。

## 主流程

```
① 品牌档案 → ② 内容诊断 → ③ 内容生成 → ④ 内容库 → ⑤ 分发计划 → ⟳ 复诊
```

1. **品牌档案**：录入品牌基础信息、目标平台、提问样本（支持账号下多品牌）。
2. **内容诊断**：向选定的 12 个平台·端（同一平台手机版 / 网页版分开统计）各提若干本地问题，得出总分、提及率、平均排名、正面率，并定位「0 提及」缺口。
3. **内容生成**：针对缺口生成 **文字 / 图片 / 视频** 三种形态内容（异步生成）。
4. **内容库**：按类型 / 状态管理产物，可逐条编辑。
5. **分发计划**：给出「发哪儿、怎么改写、配什么图」的建议；不代发，复制走到对应平台手动发布。
6. **复诊**：发布一段时间后重新诊断，对比可见度变化。

贯穿示例品牌：杭州「茶语时光」奶茶店。

## 原型预览

`docs/prototypes/` 是静态 HTML 高保真原型（Tailwind + 自定义主题），共 11 个页面（`01–10` + 总览）。

```bash
cd docs/prototypes
python3 -m http.server 4399
# 浏览器打开 http://localhost:4399/index.html
```

> 用本地静态服务器打开（而非直接双击）能保证中文文件名链接与相对资源正常加载；原型依赖 Tailwind Play CDN，需联网。

## 目录结构

```
answer-one/
├── docs/
│   ├── PRD.md              # 产品需求文档
│   ├── api-contracts.md    # 接口契约
│   ├── Plan.md             # 开发计划
│   ├── 澄清文档/            # 需求澄清记录
│   └── prototypes/         # 高保真原型（01–10 + index 总览 + assets）
├── pycore/                 # Python 后端核心骨架（API / 服务 / 集成 / 插件 / 执行）
├── .sdd/                   # 项目状态、任务、经验、日志（SDD 流程元数据）
└── AGENTS.md               # 项目轻入口
```

## 技术栈

- **原型**：HTML + Tailwind CSS（Play CDN）+ 原生 JS（共享外壳 `assets/app.js`、主题 `assets/theme.js`）。
- **后端骨架**：Python（FastAPI 风格分层：`api / core / services / integrations / plugins / execution`），需 Python 3.11+。
